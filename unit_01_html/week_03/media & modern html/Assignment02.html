Features of HTML5

Features of HTML5

  • Semantics:-HTML5 introduced new semantic elements that provide clearer meanings to different parts of a webpage. These include header, nav, section, article, aside, footer, and more.

  • Audio and Video Support:- HTML5 includes native support for embedding audio and video content without the need for plugins like Flash.

  • Canvas:-HTML5 introduced the 'canvas' element, which provides a drawing API for dynamic rendering of graphics, charts, animations, and games using JavaScript.

  • Geolocation:-HTML5 added a Geolocation API that allows websites to request a user's location information (with user permission) using GPS, Wi-Fi, or IP address.

  • Web Storage:-HTML5 introduced Web Storage APIs (localStorage and sessionStorage) that allow web applications to store data locally on the user's device. This provides a more efficient alternative to cookies for storing data.

HTML Entities

HTML Entities

HTML entities are special character codes used to represent characters that have a special meaning or cannot be easily typed or displayed directly in HTML. They are particularly useful for displaying reserved characters or characters with special formatting requirements.


Here are 5 commonly used HTML entities

  1. "<" - Represents the less-than symbol (<).

  2. ">" - Represents the greater-than symbol (>).

  3. "&" - Represents the ampersand symbol (&).

  4. """ - Represents the double quotation mark (").

  5. "©" - Represents the copyright symbol ©.

Web Accessibility

Web Accessibility

Web accessibility refers to the inclusive practice of designing and developing websites and web content in a way that ensures equal access and usability for people with disabilities. It aims to remove barriers and provide an inclusive online experience for individuals with various disabilities, including visual, auditory, physical, cognitive, and neurological impairments.


Here are some major assistive devices that play a role in providing web accessibility

  1. Keyboard Navigation:-Keyboard accessibility is essential for individuals with motor disabilities who may have difficulty using a mouse.

  2. Screen Magnifiers:-Screen magnifiers enlarge the content displayed on the screen, making it easier for individuals with low vision to read and interact with web content.

  3. Screen Readers:-Screen readers are software applications that read out the content of a webpage or computer screen aloud.

  4. Alternative Text (Alt Text) Readers:-Alt text is a textual description added to images on a webpage.

Benefits of using semantic tags
  1. Accessibility:-Semantic tags provide meaningful structure to web content, which improves accessibility for users with disabilities.

  2. Search Engine Optimization (SEO):-Search engines prioritize semantic markup when indexing and ranking webpages.

  3. Code Readability and Maintainability:-Semantic tags make the HTML code more human-readable and self-explanatory. By using tags like 'header, nav, main, article, and footer'

  4. Consistency and Standardization:-Semantic tags encourage a standardized approach to structuring web content. By following best practices and using the appropriate semantic elements, you ensure consistency across your webpages and provide a familiar experience to users

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating webpages. It provides the structure and semantics of web content.

What is Semantic HTML?

Semantic HTML refers to the use of HTML elements that convey meaning beyond just formatting. It focuses on using elements that describe the structure and purpose of the content rather than just the appearance.

Commonly Used Semantic Tags

  • <header>: Represents the introductory content or a container for a group of introductory content.
  • <nav>: Represents a section of a page that contains navigation links.
  • <main>: Represents the main content of a webpage.
  • <section>: Represents a standalone section that is semantically related to the overall content.
  • <article>: Represents a self-contained composition that can be independently distributed or syndicated.
  • <footer>: Represents the footer of a webpage or a section.
